# Kafka-Lake-Writer 需求文档

## 一、项目背景

现有开源数据同步组件（Kafka Connect、SeaTunnel、Gobblin等）在处理多Topic、动态Schema、动态路径映射场景时存在以下痛点：

- 每个Topic需独立配置Connector/Task，管理成本高
- Schema变更需重启组件，影响数据时效性
- 新增/移除Topic需手动配置并重启，无法动态生效
- 写入路径与Topic强绑定，灵活性差
- 配置变更代价高，存在停服和丢数据风险

因此需要设计一个**轻量级、高性能**的Kafka数据同步组件，直接消费Kafka数据写入HDFS/OSS存储。

## 二、功能需求

### 2.1 核心功能

| 编号   | 功能        | 描述                                  |
| ---- | --------- | ----------------------------------- |
| F-01 | Kafka消费   | 高性能消费Kafka JSON数据，支持多Topic并行消费      |
| F-02 | 多格式写入     | 支持Parquet、CSV写入格式，可按Topic独立配置       |
| F-03 | 多存储支持     | 支持HDFS和阿里云OSS（S3兼容协议）作为写入目标         |
| F-04 | 动态Topic管理 | 支持运行时新增/移除/暂停Topic，无需重启             |
| F-05 | 动态Schema  | 支持运行时变更Topic的Schema定义，自动切换          |
| F-06 | 动态路径      | 支持路径模板，根据消息字段动态路由到不同目录              |
| F-07 | 文件大小控制    | 通过行数/字节数/时间三重阈值控制文件大小，避免小文件         |
| F-08 | 容错恢复      | 进程crash后自动恢复，保证数据不丢失（At-Least-Once） |
| F-09 | 多节点部署     | 支持多实例水平扩展，自动负载均衡                    |
| F-10 | 监控告警      | 暴露Prometheus指标，支持消费延迟、写入吞吐等监控       |

### 2.2 管理功能

| 编号   | 功能       | 描述                         |
| ---- | -------- | -------------------------- |
| M-01 | REST管理接口 | 提供Topic配置的CRUD REST API    |
| M-02 | 配置热更新    | 配置变更30秒内自动生效               |
| M-03 | 运行状态查询   | 查询各Topic消费进度、Buffer状态、写入统计 |

## 三、性能需求

| 指标 | 目标值 | 条件 |
|------|--------|------|
| 消费吞吐 | >= 60,000 records/s | 单节点4C8G，JSON消息平均500B |
| 写入延迟 | <= 10分钟 | 从Kafka消费到文件可见的最大延迟 |
| 文件大小 | 128MB - 512MB | 单个Parquet文件大小范围 |
| 启动恢复时间 | <= 30秒 | 从crash到恢复正常消费 |
| 配置生效延迟 | <= 30秒 | 从配置变更到生效 |

## 四、非功能需求

| 类别 | 需求 |
|------|------|
| 数据保证 | At-Least-Once语义，不丢数据，可接受少量重复 |
| 可用性 | 支持多节点部署，单节点故障不影响整体 |
| 部署方式 | Docker容器化，支持K8s部署 |
| 资源限制 | 单节点4C8G |
| 依赖限制 | 不依赖Spark、Flink等大数据计算引擎 |

## 五、环境约束

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 1.8 | 生产环境JDK版本，代码需兼容Java 8语法 |
| Kafka | 1.x / 2.x | 生产Kafka集群版本，kafka-clients需向下兼容 |
| HDFS | 3.0.0-cdh6.3.1 | CDH发行版，使用CDH对应的hadoop-client依赖 |
| MySQL | 5.7+ / 8.0+ | 配置中心存储 |

## 六、其他约束

- 数据格式：Kafka消息体为JSON格式
- 存储协议：HDFS通过Hadoop FileSystem API，OSS通过S3兼容协议或JindoFS
- 存储路径：每个Topic可自定义HDFS/OSS路径，路径支持动态日期追加（如 `/hdfs/kafka/data/dev_stats/2026-03-21`），按天增量接入
- 配置中心：MySQL存储 + Web可视化管理界面（支持Topic/Schema的增删改查、消费状态监控）
- 监控：Prometheus + Grafana
